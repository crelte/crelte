# Crelte Docs

The documentation site is built with [VitePress](https://vitepress.dev) and
lives in this directory. The API reference is generated with
[TypeDoc](https://typedoc.org) from the source of `packages/crelte`.

## Setup

The docs are a **standalone package**, intentionally not part of the npm
workspace. They have their own `package.json` and `package-lock.json`, so
vitepress (and its vite version) can be upgraded independently of the other
packages.

This means the root `npm ci` / `npm install` does **not** install the docs
dependencies. Install them once with:

```bash
# from the repository root
npm run docs:install

# or directly
cd docs && npm install
```

Since the API reference is generated from the built packages, the workspace
needs to be installed and built as well:

```bash
# from the repository root
npm ci
npm run build
```

## Development

```bash
# from the repository root
npm run docs:dev

# or directly
cd docs && npm run dev
```

This first runs typedoc (generating the API reference into `docs/types/`)
and then starts the vitepress dev server.

> Note: changes to `packages/crelte` only show up in the API reference after
> rebuilding the package (`npm run build`) and restarting the dev server.

## Build & Preview

```bash
npm run docs:build    # build the static site into docs/.vitepress/dist
npm run docs:preview  # serve the built site locally
```

Deployment to GitHub Pages happens automatically on pushes to `main` via
`.github/workflows/docs.yaml`.

## Structure

Crelte has 4 different kinds of documentation:

1. **Tutorial** (`tutorial/`): an onboarding which should help you build a
   simple blog website.
2. **Docs** (`docs/`): the textual documentation which gives examples and
   describes how the whole thing works. This should be read by people who
   want to fully understand crelte and what features it has. It should also
   be a reference to quickly look up "oh, how does that work".
3. **Cookbook** (`cookbook/`): useful examples and code snippets which are a
   good solution and example for common problems like language navigation,
   pagination, filters etc.
4. **Types** (`types/`, auto-generated by typedoc, not committed): the API
   reference, to look up "how was this method called again".
